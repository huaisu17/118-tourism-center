# 118 旅游智慧中心 UI 改造方案

## 目标

在现有 `118 旅游智慧中心` 项目基础上，改造成更接近参考图的“科技感综合指挥大屏”风格，做到：

- 保留现有数据结构、ECharts 初始化逻辑和大部分功能模块
- 重做页面布局层级和视觉样式
- 统一卡片、标题、地图、图表的设计语言
- 尽量避免整套推翻重写，优先采用低风险改造方式

## 现状判断

当前项目本身已经具备比较接近目标图的基础结构：

- 左中右三大区域已经存在
- 中间已有地图主视觉
- 两侧已有多个数据卡片
- 图表均已通过 ECharts 初始化

因此这次改造不建议从零新建页面，而建议采用：

`保留数据与图表逻辑，重做页面骨架 + 卡片皮肤 + 地图视觉 + 图表主题`

## 推荐改造思路

### 1. 保留哪些内容

以下内容建议继续沿用：

- `static/index.js` 中的图表初始化逻辑
- `static/chart.js` 中各图表的数据结构和大部分 option 逻辑
- 原有模块的功能划分
- 原有自适应 rem 方案

这样做的好处是：

- 改造成本低
- 不容易把数据绑定改坏
- 可以先快速把页面“长相”改出来

### 2. 重点重做哪些内容

重点改造四部分：

1. 页面布局骨架
2. 卡片视觉皮肤
3. 中间地图主舞台
4. ECharts 图表主题

## 一、页面布局改造

### 当前问题

当前页面结构是：

- 顶部 `header`
- 中部 `content`
- 底部 `footer`

这种上下切分方式更像传统数据看板，不像参考图那种“中心主舞台 + 两侧信息墙”的指挥中心布局。

### 推荐结构

建议改成单屏三列主布局：

- 顶部：发光标题栏
- 左侧：2 列小卡片区
- 中间：大地图主视觉区
- 右侧：2 列小卡片区

推荐结构示意：

```html
<div class="dashboard">
  <header class="topbar">...</header>
  <main class="screen">
    <section class="panel panel-left">...</section>
    <section class="stage">...</section>
    <section class="panel panel-right">...</section>
  </main>
</div>
```

### 布局建议

- 左侧宽度：约 `26%`
- 中间宽度：约 `48%`
- 右侧宽度：约 `26%`
- 卡片之间统一使用 `gap`
- 不再单独保留高占比 `footer`
- 底部原表格和趋势图拆分后并入左右栏位

### 模块重排建议

- 左上：实时游客流量
- 左中：车流分析
- 左下：满意度 / 核心指标
- 左下补充：今日累计车流
- 中间：地图主视觉
- 右上：停车场数据
- 右中：游客消费占比
- 右下：门票趋势
- 右下补充：12301 呼叫 / 游客趋势表格

## 二、卡片样式改造

### 当前问题

当前 `.item` 卡片样式较基础：

- 只是半透明蓝底
- 边框简单
- 标题缺少科技装饰
- 卡片之间视觉层次不够统一

### 目标风格

参考图的卡片特征：

- 深蓝渐变背景
- 青蓝色发光边框
- 上侧带切角或折线装饰
- 标题有独立的装饰条
- 卡片边缘有轻微内阴影和高光

### 推荐变量

```css
:root {
  --bg: #07162d;
  --panel: rgba(10, 36, 78, 0.72);
  --panel-strong: rgba(8, 27, 60, 0.9);
  --panel-border: rgba(58, 181, 255, 0.45);
  --cyan: #35d8ff;
  --cyan-soft: #7be7ff;
  --gold: #f7c35f;
  --text: #d9f3ff;
  --text-dim: #7fa9c9;
}
```

### 推荐卡片样式方向

- 背景使用深蓝渐变，不要纯色
- 边框颜色统一为青蓝色弱发光
- 卡片上方增加标题装饰线
- 卡片圆角不要太大，偏硬朗
- 内部使用更多留白，不要内容贴边

### 推荐标题样式方向

- 标题文字统一为亮青色
- 标题字号统一
- 使用伪元素做两端切角
- 可增加细线和短横条作为科技感装饰

## 三、中间地图区域改造

### 当前问题

当前中间区域是：

- 一张静态地图图像
- 几个 pulse 圆点
- 一个简单拥挤度图例

和目标图相比，差距主要在于：

- 地图边界不够发光
- 缺少主舞台氛围
- 缺少区域标签系统
- 缺少旁侧浮动榜单

### 推荐升级方向

#### 方案 A：低成本快速改造

继续使用现有地图图片，但增强视觉：

- 地图容器加蓝色外发光
- 增加底部圆形能量底盘
- 增加纵向光柱和粒子背景
- 点位标签改成“数值 + 名称”形式
- 右侧增加悬浮排行卡片

#### 方案 B：中成本高还原

将地图改成 ECharts 地图或 SVG 地图：

- 使用区域边界高亮
- 每个区域支持标签
- 支持 hover 高亮和涟漪点位
- 更接近参考图的轮廓感

### 地图区推荐增加的元素

- 地图顶部辅助标题
- 地图区域数值标签
- 底部科技底盘
- 右侧悬浮榜单
- 背景星点/粒子层
- 边界描边发光

## 四、图表主题改造

### 当前问题

目前各图表风格不够统一：

- 配色偏旧
- 有些图表颜色过多
- 线条、柱体、标签风格不一致
- 有些图表更像模板示例，不像同一套系统

### 统一改造原则

所有图表都建议收敛到一套主题：

- 主色：青蓝
- 强调色：金黄
- 风险色：橙红
- 网格线：低对比蓝灰
- 坐标文字：浅蓝灰
- 数据文字：亮青或白色

### 各类图表建议

- 柱状图：细柱、青蓝渐变、柱顶高亮
- 折线图：细线、面积渐变、末端点高亮
- 环图 / 饼图：发光环、中间大数字、外圈描边
- 排行图：横向发光条形图，头部 3 项强调色区分

## 五、视觉风格统一建议

### 背景

不要只保留一张平面背景图，建议增加：

- 深色径向渐变底
- 局部光晕
- 微弱网格或电路线纹理
- 星点粒子层

### 字体

如果项目允许，可考虑：

- 标题：更硬朗的科技风字体
- 正文：继续使用微软雅黑或无衬线字体

如果不额外引字体，也建议通过字重、字距、阴影来提升标题气质。

### 配色建议

尽量控制在三种主角色：

- 青蓝：主视觉、标题、边框、主要图表
- 金黄：重点数字、预警、关键状态
- 深蓝：背景、卡片底色

避免紫色、粉色过多混用，以及一张图里同时出现太多高饱和色。

## 六、建议的实际落地顺序

### 第一步：先处理基础问题

- 确认所有文件编码统一为 UTF-8
- 修正文案乱码
- 先把页面标题、模块标题改成正常中文

### 第二步：重做 HTML 骨架

重点修改 `index.html`：

- 调整模块分组
- 改成新的三列主布局
- 预留地图区浮层和标题装饰节点

### 第三步：整体重写样式

重点修改 `static/style.css`：

- 页面背景
- 顶部标题
- 卡片皮肤
- 标题装饰
- 地图区氛围层

### 第四步：统一图表主题

重点修改 `static/chart.js`：

- 柱图渐变
- 线图配色
- 坐标轴样式
- 标签样式
- 环图配色与中心文案

### 第五步：细节收尾

- 调整间距
- 对齐文字
- 校准地图标签位置
- 检查大屏分辨率适配

## 七、最推荐的实现策略

如果目标是“尽快做出接近参考图的版本”，最推荐的策略是：

### 保留

- `static/index.js`
- 数据变量
- 现有 ECharts 初始化流程

### 重写

- `index.html` 的模块结构
- `static/style.css` 的绝大多数视觉样式

### 局部微调

- `static/chart.js` 中各图表的颜色、边距、字体和渐变

这种方式的优点：

- 见效最快
- 风险最小
- 后续继续精修也方便

## 八、文件级改造建议

### 1. `index.html`

负责：

- 页面整体结构重排
- 卡片区域重新分组
- 中间地图区增加装饰层

### 2. `static/style.css`

负责：

- 页面整体背景
- 卡片视觉
- 标题视觉
- 地图主舞台氛围
- 响应式与间距调整

### 3. `static/chart.js`

负责：

- 图表主题统一
- 配色和渐变调整
- label、axis、grid 的风格统一

### 4. `static/index.js`

负责：

- 保持初始化逻辑不变
- 必要时补充新的 DOM 初始化

## 九、最终建议

这次改造最关键的不是“多做几个图”，而是把下面四件事统一起来：

- 布局骨架统一
- 卡片皮肤统一
- 地图主视觉强化
- 图表主题统一

只要这四件事做好，即使保留现有数据和模块内容，整体观感也会非常接近参考图。

## 参考实施优先级

高优先级：

- 页面三列重排
- 卡片皮肤重做
- 顶部标题重做
- 地图主视觉升级

中优先级：

- 图表统一主题
- 排行榜和浮层补充
- 表格重做成更轻量的视觉模块

低优先级：

- 粒子动画
- 细微发光特效
- hover 交互动效

## 建议结论

不要在当前页面上零碎修补，而要以现有项目为底座，做一次“结构不推翻、视觉全面升级”的改造。

这是当前成本、效果、可维护性之间最平衡的方案。
