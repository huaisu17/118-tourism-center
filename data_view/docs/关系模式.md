# 智校数据综合展示中心 — 关系模式

> 由 ER 图转换而来的关系模式，可直接用于数据库设计。

---

## 转换规则

| ER 元素 | 关系模式处理 |
|---------|-------------|
| 实体 | 转换为一个**关系** |
| 实体属性 | 转换为关系的**属性** |
| 实体主键 | 作为主键（不加特殊标记，由上下文推断） |
| 1:n 关系 | 在"n"端加入"1"端的**外键(FK)** |
| n:m 关系 | 转换为一个**独立的关系**，包含双方外键 + 关系属性 |
| 三元关系 | 转换为一个**独立的关系**，包含三方外键 + 关系属性 |
| 关系属性 | 随 n:m 关系或三元关系一起放入关联关系中 |

---

## 关系模式

### 一、基础档案关系

**1）区域（行政区域）**

区域（**区域ID**，区域名称，区域编码，父区域ID(FK)）

> 父区域ID 外键引用：区域(区域ID)，用于树形结构

---

**2）学校**

学校（**学校ID**，区域ID(FK)，学校名称，学校编码，学校类型，地址，学生人数，教师人数，联系人，联系电话，状态）

> 区域ID 外键引用：区域(区域ID)

---

**3）食堂**

食堂（**食堂ID**，学校ID(FK)，食堂名称，食堂编码，食堂类型，负责人，容纳人数，状态）

> 学校ID 外键引用：学校(学校ID)

---

**4）供应商**

供应商（**供应商ID**，供应商名称，供应商编码，联系人，联系电话，地址，营业执照号，综合评分，评级，状态）

---

**5）食材分类**

食材分类（**分类ID**，分类名称，分类编码，父分类ID(FK)）

> 父分类ID 外键引用：食材分类(分类ID)，用于树形结构

---

**6）食材**

食材（**食材ID**，分类ID(FK)，食材名称，食材编码，规格，计量单位，储存条件，状态）

> 分类ID 外键引用：食材分类(分类ID)

---

### 二、采购相关关系

**7）采购订单**

采购订单（**采购ID**，学校ID(FK)，食堂ID(FK)，供应商ID(FK)，订单编号，采购日期，总金额，已支付金额，待支付金额，支付状态，支付日期，备注）

> 学校ID 外键引用：学校(学校ID)  
> 食堂ID 外键引用：食堂(食堂ID)  
> 供应商ID 外键引用：供应商(供应商ID)

---

**8）采购明细（n:m 关联关系）**

采购明细（**明细ID**，采购ID(FK)，食材ID(FK)，数量，单价，金额）

> 采购ID 外键引用：采购订单(采购ID)  
> 食材ID 外键引用：食材(食材ID)

> 说明：由 ER 图中"采购订单 **n:m** 采购 食材"关系转换而来，包含关系属性"数量、单价、金额"。

---

**9）食材价格**

食材价格（**记录ID**，食材ID(FK)，记录日期，价格，地区，数据来源）

> 食材ID 外键引用：食材(食材ID)

---

**10）供应商食堂食材关联（三元关联关系）**

供应商食堂食材关联（**关联ID**，供应商ID(FK)，食堂ID(FK)，食材ID(FK)，供应单价，是否主供）

> 供应商ID 外键引用：供应商(供应商ID)  
> 食堂ID 外键引用：食堂(食堂ID)  
> 食材ID 外键引用：食材(食材ID)

> 说明：由 ER 图中"供应商 + 食堂 + 食材 **三元** 供应"关系转换而来，包含关系属性"供应单价、是否主供"。表示某个供应商向某个食堂供应某种食材。

---

### 三、管控相关关系

**11）日管控**

日管控（**记录ID**，学校ID(FK)，食堂ID(FK)，管控日期，排查总数，已完成数，待整改数，状态，检查人）

> 学校ID 外键引用：学校(学校ID)  
> 食堂ID 外键引用：食堂(食堂ID)

---

**12）管控明细**

管控明细（**明细ID**，管控ID(FK)，检查项，结果，问题描述，整改状态，整改时间）

> 管控ID 外键引用：日管控(记录ID)

> 说明：由 ER 图中"日管控 **1:n** 包含 管控明细"关系转换而来，在管控明细端加入日管控的外键。

---

**13）周排查**

周排查（**记录ID**，学校ID(FK)，排查周次，排查总数，合格数，黄线问题，基础问题，合格率，状态）

> 学校ID 外键引用：学校(学校ID)

---

**14）月调度**

月调度（**记录ID**，学校ID(FK)，食堂ID(FK)，月份，调度名称，状态，调度内容，详情页URL，页面编码，调度时间）

> 学校ID 外键引用：学校(学校ID)  
> 食堂ID 外键引用：食堂(食堂ID)

---

### 四、业务记录关系

**15）订餐记录**

订餐记录（**记录ID**，学校ID(FK)，食堂ID(FK)，订餐日期，餐次，菜品，学生订餐数，教师订餐数，总订餐数，总金额，营养类别，环比变化率，排名）

> 学校ID 外键引用：学校(学校ID)  
> 食堂ID 外键引用：食堂(食堂ID)

---

**16）食材验收**

食材验收（**记录ID**，学校ID(FK)，食堂ID(FK)，食材ID(FK)，验收日期，批次号，数量，质量状态，合格率，验收人，备注）

> 学校ID 外键引用：学校(学校ID)  
> 食堂ID 外键引用：食堂(食堂ID)  
> 食材ID 外键引用：食材(食材ID)

---

**17）师生评价**

师生评价（**记录ID**，学校ID(FK)，食堂ID(FK)，评价日期，评价者类型，评分，评价内容，评价菜品）

> 学校ID 外键引用：学校(学校ID)  
> 食堂ID 外键引用：食堂(食堂ID)

---

**18）学生营养**

学生营养（**记录ID**，学校ID(FK)，食堂ID(FK)，食材ID(FK)，记录日期，营养类别，摄入量，蛋白质，脂肪，碳水化合物，热量）

> 学校ID 外键引用：学校(学校ID)  
> 食堂ID 外键引用：食堂(食堂ID)  
> 食材ID 外键引用：食材(食材ID)

---

**19）消费记录**

消费记录（**记录ID**，学校ID(FK)，食堂ID(FK)，月份，消费金额，消费人次，人均消费，区域名称）

> 学校ID 外键引用：学校(学校ID)  
> 食堂ID 外键引用：食堂(食堂ID)

---

## ER 图 → 关系模式 转换对照

| ER 图关系 | 关系模式处理 |
|-----------|-------------|
| 区域 **1:n** 包含 学校 | 学校表中加入 区域ID(FK) |
| 学校 **1:n** 拥有 食堂 | 食堂表中加入 学校ID(FK) |
| 食材分类 **1:n** 分类 食材 | 食材表中加入 分类ID(FK) |
| 供应商 **1:n** 供货 采购订单 | 采购订单表中加入 供应商ID(FK) |
| 食堂 **1:n** 产生 采购订单 | 采购订单表中加入 食堂ID(FK) |
| 食堂 **1:n** 产生 日管控 | 日管控表中加入 食堂ID(FK) |
| 学校 **1:n** 产生 日管控 | 日管控表中加入 学校ID(FK) |
| 日管控 **1:n** 包含 管控明细 | 管控明细表中加入 管控ID(FK) |
| 学校 **1:n** 产生 周排查 | 周排查表中加入 学校ID(FK) |
| 学校 **1:n** 产生 月调度 | 月调度表中加入 学校ID(FK) |
| 食堂 **1:n** 产生 月调度 | 月调度表中加入 食堂ID(FK) |
| 学校 **1:n** 产生 订餐记录 | 订餐记录表中加入 学校ID(FK) |
| 食堂 **1:n** 产生 订餐记录 | 订餐记录表中加入 食堂ID(FK) |
| 学校 **1:n** 产生 食材验收 | 食材验收表中加入 学校ID(FK) |
| 食堂 **1:n** 产生 食材验收 | 食材验收表中加入 食堂ID(FK) |
| 食材 **1:n** 验收 食材验收 | 食材验收表中加入 食材ID(FK) |
| 学校 **1:n** 产生 师生评价 | 师生评价表中加入 学校ID(FK) |
| 食堂 **1:n** 产生 师生评价 | 师生评价表中加入 食堂ID(FK) |
| 学校 **1:n** 产生 学生营养 | 学生营养表中加入 学校ID(FK) |
| 食堂 **1:n** 产生 学生营养 | 学生营养表中加入 食堂ID(FK) |
| 食材 **1:n** 摄入 学生营养 | 学生营养表中加入 食材ID(FK) |
| 学校 **1:n** 产生 消费记录 | 消费记录表中加入 学校ID(FK) |
| 食堂 **1:n** 产生 消费记录 | 消费记录表中加入 食堂ID(FK) |
| 食材 **1:n** 波动 食材价格 | 食材价格表中加入 食材ID(FK) |
| 供应商 + 食堂 + 食材 **三元** 供应 | 独立关系：供应商食堂食材关联(供应商ID, 食堂ID, 食材ID, 供应单价, 是否主供) |
| 采购订单 **n:m** 采购 食材 | 独立关系：采购明细(采购ID, 食材ID, 数量, 单价, 金额) |
