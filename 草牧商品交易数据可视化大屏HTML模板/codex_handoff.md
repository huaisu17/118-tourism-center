### 2. 用户手工调过位置

用户自己手动调过：

- `入驻动态` 标题位置
- `入驻动态` 列表位置
- 环图位置

所以不要默认 CSS 还是“纯助手改的原始状态”。  
改样式前先读当前文件，别覆盖用户手动改动。

### 3. PowerShell 乱码不代表文件坏了

这个项目文件本身是 UTF-8。  
之前很多乱码只是终端读取方式不对。

稳定做法：

```powershell
[Console]::OutputEncoding = [System.Text.Encoding]::UTF8
[System.IO.File]::ReadAllText('path', [System.Text.Encoding]::UTF8)
```

以后读文件优先用 UTF-8 显式读取。

### 4. 很多图有随机刷新

这个模板原本大量用了 `setInterval` 做演示数据刷新。  
如果只改初始值，不去掉这些定时器，几秒后页面就会自己变回乱跳状态。

因此：

- 如果用户要固定展示 `goal.png` 风格数据
- 就必须去掉这些随机刷新

本次已经去掉了一部分关键块的随机刷新。

### 5. 不要轻易动全局样式

这个模板有不少公共样式是整屏复用的。  
之前改错过一次，把右上整块的背景框误删到外层容器上。

以后原则：

- 优先改局部容器
- 不要随便从公共规则里删整个模块
